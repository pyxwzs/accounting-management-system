# 毕设需求进度

> 说明：每完成一条需求，在本文件中更新对应状态与简要说明。

---

## 1. 丰富「我的预算」模块

**状态：已完成**

- 支持设置预算金额与统计周期（按月 / 按季 / 按年 / 不按周期），用于例如「月可支配 6000」等场景。
- 「我的预算」页以饼图展示已使用、剩余额度及超支部分。
- 达到预警比例时，在页面以提示条展示；周期型预算在每日 9:00 对注册邮箱发送预警邮件（需配置邮箱且用户填写邮箱）。
- 超支或上周期超支时，在页面给出下一周期预算与日均消费建议。
- 数据库：`sys_budget` 含 `cycle_type`、`warn_ratio`、`last_email_date` 等字段；完整脚本见 `database/obt_db.sql`。

## 2. 收件箱模块：将系统公告和意见反馈归纳到收件箱

**状态：已完成**

- 新增前台页面 `/inbox`（收件箱），`el-tabs` 分为「系统公告」与「意见反馈」。
- 公告列表沿用 `/notice/notices/` 分页数据，标题跳转 `/notice/:id` 查看详情；详情页面包屑可返回收件箱。
- 意见反馈仅展示 **当前登录用户** 的留言及回复（请求带 `user_id`，后端已改为 `eq` 精确匹配）；支持加载更多与提交新反馈（原留言接口）。
- 顶栏与页脚原「系统公告」「意见反馈/留言板」合并为 **收件箱**；`/notice`、`/leaveword` 重定向至 `/inbox` 对应 Tab（`?tab=notice` / `?tab=feedback`）。

## 3. 丰富「存钱计划」模块

**状态：已完成**

- 列表展示 **存钱进度**（已存/目标，`el-progress`）与 **时间进度**（已过天数/计划总天数）。
- **完成预测**：汇总当前用户近 **30 天**「收支记录」中「收入」「支出」，计算日均结余 `(收入−支出)/30`，与「剩余目标 ÷ 剩余天数」所需日均存入比较，生成文案与「有望按期 / 存在风险」标签；无收支记录时提示无法预测。
- 后端：`PlanForecastHelper` + `InfoService.sumIncomeExpenseBetween`；`Plan` 实体增加若干 `@TableField(exist = false)` 字段；分页查询后自动填充。计划列表按 `user_id` 精确查询已改为 `eq`。

## 4. 丰富「财经资讯」模块

**状态：已完成**

- **实时财经新闻**：后台按 `finance.rss.url` 拉取 RSS，去重写入 `sys_article`（`source_url`）；前台登录用户可点「同步最新资讯」触发入库。
- **跳转原文**：列表与详情展示「阅读原文」（`source_url` 外链）；后台文章表单可选填原文链接。
- **收藏资讯**：`sys_article_favorite` 存用户收藏；列表/详情展示星标，接口 `POST /article/favorite/toggle`；需登录。表结构见 `database/obt_db.sql`。

---

## 5. 新增日志功能

**状态：已完成**

- **操作审计**：通过 AOP（`LogAspect`）拦截带 `@Operation` 的 Controller 方法，将操作说明、操作人（从 JWT 解析）、成功/失败及失败原因写入 `sys_log`；无 Token 或 Token 无效时操作人记为「未知用户」。
- **覆盖范围**：各模块增删改、用户注册与密码相关、文件上传、资讯 RSS 同步与收藏切换、删除日志等均打标记录；分页查询接口不打标。
- **后台查询**：沿用「操作日志」页（`/system/log`）分页与筛选；列表「操作类型」支持模糊匹配（`operation` 字段 `like`）。
- **菜单**：「操作日志」菜单与权限已包含在 `database/obt_db.sql` 的初始化数据中。

## 6. 收支统计可视化改版（前台「收支记录」`/charts`）
